## Overview

The personal website now has a **Colophon** page at [`/colophon`](https://patrickfoster.dev/colophon) that documents how the site itself is put together — the stack, the hosting, and the principles guiding it.

The page is rendered by `src/components/Colophon.js` and styled by `src/components/colophon.css`. Content is defined in a local `sections` array inside the component, so edits are a single-file change.

## What changed

A new route was added to `src/App.js`:

```js
<Route exact path='/colophon' component={Colophon} />
```

This makes `/colophon` reachable from anywhere on the site. The page also includes a "← Back home" link to `/` at the top.

## Editing the page

To update the colophon:

1. Open `src/components/Colophon.js`.
2. Edit the `sections` array to add, remove, or reword bullets. Each section is `{ heading: string, items: string[] }`.
3. Update the `updatedAt` constant (currently `'June 2026'`) when content meaningfully changes.
4. Adjust copy in the `colophonIntro` paragraph if the framing of the page shifts.

Styling lives in `src/components/colophon.css` and is scoped via the `.colophon`, `.colophonHeader`, `.colophonMeta`, `.colophonIntro`, and `.colophonSection` class names.

## Related routes

The colophon joins the existing set of routes registered in `src/App.js`: `/`, `/blog`, `/blog/:id`, `/timeline`, `/uses`, `/now`, `/reading-list`, `/talks`, and `/magic-the-gathering`. Unmatched paths fall through to the `NotFound` component.
