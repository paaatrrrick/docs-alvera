ナビゲーションメニューは、すべての Web サイトに表示されるリンクの一覧です。

新しいページを追加するたびに `docs.json` を更新することになります。ページは自動では表示されません。

## ナビゲーション構文

ナビゲーション構文は再帰的なので、ネストしたナビゲーショングループを作成できます。ページ名に `.mdx` を含める必要はありません。

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## フォルダ

MDX ファイルをフォルダに入れ、`docs.json` のパスを更新するだけです。

たとえば、`https://yoursite.com/your-folder/your-page` にページを配置するには、`your-folder` というフォルダを作成し、その中に `your-page.mdx` という MDX ファイルを置きます。

<Warning>

別のフォルダ内にネストしない限り、フォルダ名として `api` を使用することはできません。Mintlify は Next.js を使用しており、トップレベルの `api` フォルダは内部のサーバー呼び出し用に予約されています。`api-reference` のようなフォルダ名であれば受け入れられます。

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## 非表示のページ

`docs.json` に含まれていない MDX ファイルはサイドバーに表示されませんが、検索バーや直接リンクからアクセスできます。
