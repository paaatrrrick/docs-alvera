<Info>
  **前提条件**:
  - Node.js バージョン 19 以上
  - `docs.json` ファイルを含むドキュメントリポジトリ
</Info>

お使いの OS に Mintlify をインストールして実行するには、次の手順に従ってください。

<Steps>
<Step title="Mintlify CLI をインストールする">

```bash
npm i -g mint
```
</Step>

<Step title="ローカルでプレビューする">

`docs.json` ファイルがあるドキュメントディレクトリに移動し、次のコマンドを実行します:

```bash
mint dev
```

ドキュメントのローカルプレビューが `http://localhost:3000` で利用できるようになります。

</Step>
</Steps>

## カスタムポート

デフォルトでは、Mintlify はポート 3000 を使用します。`--port` フラグを使用して、Mintlify が実行されるポートをカスタマイズできます。たとえば、ポート 3333 で Mintlify を実行するには、次のコマンドを使用します:

```bash
mint dev --port 3333
```

すでに使用中のポートで Mintlify を実行しようとすると、次に利用可能なポートが使用されます:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify のバージョン

各 CLI リリースは、特定の Mintlify のバージョンに対応していることに注意してください。ローカルプレビューが本番バージョンと一致しない場合は、CLI を更新してください:

```bash
npm mint update
```

## リンクの検証

CLI を使ってドキュメント内のリンクを検証できます。壊れたリンクを特定するには、次のコマンドを使用します:

```bash
mint broken-links
```

## コードフォーマット

MDX を認識・フォーマットするために、IDE の拡張機能を使うことをおすすめします。VSCode ユーザーの場合は、シンタックスハイライト用に [MDX VSCode 拡張機能](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx)、コードフォーマット用に [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) を検討してください。

## トラブルシューティング

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    これは node のバージョンが古いことが原因の可能性があります。次を試してください:
    1. 現在インストールされている CLI を削除します: `npm remove -g mint`
    2. Node v19 以上にアップグレードします。
    3. CLI を再インストールします: `npm i -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">
  
    解決策: デバイスのルートに移動し、`~/.mintlify` フォルダを削除します。その後、再度 `mint dev` を実行してください。
  </Accordion>
</AccordionGroup>

最新の CLI バージョンの変更点が気になる方は、[CLI チェンジログ](https://www.npmjs.com/package/mintlify?activeTab=versions)をご覧ください。
