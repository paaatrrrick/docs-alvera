Mintlify は、MDX ファイルの Git リポジトリを高速で検索可能なドキュメントサイトに変換します。このページでは、内部で何が起きているかを把握できるよう、構成要素を解説します。

## コミットから公開ページまで

1. **コンテンツを書く。** ページは MDX で執筆します。MDX は Markdown に `<Card>`、`<Tabs>`、`<Accordion>` などの React コンポーネントが加わったものです。
2. **ナビゲーションを設定する。** `docs.json` でタブ、グループ、テーマ、ナビバーやフッターなどのグローバル要素を定義します。
3. **GitHub にプッシュする。** Mintlify の GitHub アプリがデプロイブランチの変更を検知します。
4. **Mintlify がサイトをビルドする。** コンテンツが検証され、リンクがチェックされ、サイトが静的アセットにコンパイルされます。
5. **サイトが公開される。** 新しいビルドが通常数秒以内にドメインへデプロイされます。

## ローカル開発

プッシュする前に変更をプレビューするには、Mintlify CLI を使用します:

```bash
mint dev
```

これにより、`http://localhost:3000` でドキュメントがホットリロード付きで配信されます。問題を早期に発見するには、次を実行してください:

```bash
mint validate
mint broken-links
```

## オーサリングモデル

- **MDX ファイル** — すべてのページは YAML フロントマター(`title`、`description`)を持つ `.mdx` ファイルです。
- **コンポーネント** — Mintlify には、コールアウト、タブ、コードグループ、アコーディオン、カードなどの組み込みコンポーネントが含まれています。
- **スニペット** — 再利用可能なコンテンツは `snippets/` に置かれ、任意のページからインポートできます。
- **画像とアセット** — リポジトリのルートからの相対パスで参照されます。

## 設定

`docs.json` はサイト構造の唯一の信頼できる情報源(single source of truth)です:

- `navigation` — タブ、グループ、ページ順序。
- `colors`、`logo`、`favicon` — ブランディング。
- `navbar`、`footer` — グローバルなリンクとソーシャル。
- `contextual` — 各ページの「Copy / Open in ChatGPT」メニューを制御します。

## 検索と AI

公開されるすべてのサイトには次が含まれます:

- **全文検索** — ビルドごとに自動的にインデックス化されます。
- **AI チャット** — ドキュメントをコンテキストに使って質問に回答します。
- **MCP と LLM インテグレーション** — Cursor や Claude などのツールがドキュメントを参照できるようにします。

## デプロイ環境

| 環境 | トリガー | 用途 |
| ---- | -------- | ---- |
| ローカル | `mint dev` | 変更の下書きとプレビュー |
| プレビュー | プルリクエスト | マージ前の変更レビュー |
| 本番 | デプロイブランチへのマージ | 公開サイト |
