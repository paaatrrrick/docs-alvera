## Overview

The `personalwebsite` repo is Patrick Foster's personal site, deployed at [patrickrfoster.com](https://www.patrickrfoster.com). It's a single-page React app bootstrapped with Create React App and routed with `react-router-dom`.

## Stack

- **Framework:** React 18 (Create React App via `react-scripts` 5)
- **Routing:** `react-router-dom` v4 (`Switch` + `Route`)
- **Hosting:** Netlify (`netlify` build script with `_redirects` in `public/`)
- **Styling:** Per-component CSS files (no CSS framework)

## Routes

Defined in `src/App.js`:

| Path | Component |
| --- | --- |
| `/` | `Home` |
| `/blog` | `Blog` |
| `/blog/:id` | `ViewBlog` |
| `/timeline` | `Timeline` |
| `/uses` | `Uses` |
| `/now` | `Now` |
| `/reading-list` | `ReadingList` |
| `/talks` | `Talks` |
| `/magic-the-gathering` | `Haus` |
| _fallback_ | `NotFound` |

## Home page

`src/Home.js` composes the landing experience:

- Animated hero with "Hello, I'm Patrick" headings
- Work experience list rendered from `constants/work.js` (collapsible вЂ” shows 2, expands to all)
- Projects grid rendered from `constants/projects.js` (same collapse pattern)
- Testimonials, contact section with click-to-copy email, and footer

## Content model

Most page content is data-driven. To update the site you typically edit a file under `src/constants/` rather than a component:

- `work.js` вЂ” work experience entries
- `projects.js` вЂ” project cards
- `reading.js` вЂ” reading list
- `talks.js` вЂ” talks
- `testimonials.js` вЂ” testimonials
- `articles.js` + `articles/` вЂ” blog posts
