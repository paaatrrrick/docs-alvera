This glossary defines terms you'll encounter across the documentation. Use it as a quick reference when something is unfamiliar.

## MDX

A file format that combines [Markdown](https://www.markdownguide.org/) with JSX, allowing you to embed React-style components directly in your content. All documentation pages in this site are authored as `.mdx` files with YAML frontmatter.

## Frontmatter

The block of YAML at the top of an MDX file, fenced by `---`, that sets page metadata such as `title` and `description`. Mintlify uses frontmatter to render page headers, SEO tags, and navigation labels.

## Navigation

The structure that organizes pages into tabs, groups, and entries in the sidebar. Navigation is configured in [`docs.json`](/essentials/navigation) and controls how readers move through the docs.

## Component

A reusable UI building block (such as `<Card>`, `<Columns>`, or `<Note>`) that you can drop into MDX to create rich layouts without writing custom code. See [Markdown syntax](/essentials/markdown) for the full list of available components.

## Snippet

A reusable piece of content stored in `snippets/` that can be imported into multiple pages. Snippets keep shared text, code, or components in one place so updates flow everywhere they're used. Learn more in [Reusable snippets](/essentials/reusable-snippets).
