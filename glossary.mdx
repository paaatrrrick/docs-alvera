This glossary defines terms you will encounter throughout these docs. Skim it once, then refer back whenever a word is unclear.

## Terms

- **Deployment** — A published version of your site. Each push to your main branch creates a new deployment.
- **Preview** — A temporary build of a pull request, so you can review changes before they go live.
- **Navigation** — The sidebar structure that organizes your pages, defined in your `docs.json` config file.
- **Snippet** — A reusable block of content stored in `snippets/` that you can include on multiple pages.
- **Frontmatter** — The YAML block at the top of every MDX file that sets page metadata like `title` and `description`.

If you are just getting started, read the [quickstart](/quickstart) first, then set up your environment with the [local development guide](/development).
