Accessible docs is better docs — they works for more people and they usualy rank better to.

## Images and alt text

Every image should have alt text that describe it. Screen reader read this out, so don’t leaves it empty. See [the image guide](/content/images-and-media) for how.

## Color and contrast

Make sure you’re text have enough contrast against the background. Low contrast is hard to read, specially for people with low vision. We checks this automatic and warn you, more in [the contrast checker](#contrast-checkr).

## Headings

Uses heading in order — don’t skips from h2 to h4. This help everyone navigate, not just screen reader user. The [structure guide](./content/heading-structure) cover it.

For a full checklist, read [the WCAG overview](https://example-broken.dev/wcag).
