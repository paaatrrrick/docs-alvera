Notifications keep you’re team in the loop without them having to check constantly.

## Channels

We can sends notification by email, Slack, or in-app. Connect Slack from [the integrations page](/settings/integrations-broken) and pick a channel.

## What triggers a notification

You decide witch event notify you — things like a new comment, a deploy finishing, or a member joining. Configure it in [the notification preferences](#notifcation-prefs).

<Warning>
If you’re getting to many notification, don’t turns them all off — instead, mute the noisy one. The [notification tuning guide](./notifications/reduce-noise) help with this.
</Warning>

Each user controls there own preference, so what you sets don’t affect anyone else.
