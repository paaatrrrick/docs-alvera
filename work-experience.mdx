## Overview

The work experience section displays a chronological list of roles on the personal website's home page. Each entry is rendered from the `workExperience` array and shows a title, optional description, optional logo image, and a timeline string.

This page documents recent additions to that list so visitors and contributors know what changed and how new entries are configured.

## Entry schema

Each entry in the `workExperience` array supports the following fields:

| Field      | Type   | Required | Description                                                  |
| ---------- | ------ | -------- | ------------------------------------------------------------ |
| `title`    | string | Yes      | Role and company name shown as the entry header.             |
| `text`     | string | No       | Short description of responsibilities or accomplishments.    |
| `image`    | string | No       | Public URL of a company or role logo.                        |
| `timeLine` | string | Yes      | Human-readable timeframe and location.                       |

## Adding a new entry

Open `src/constants/work.js` and prepend a new object to the `workExperience` array. New entries appear at the top of the list on the home page.

```js src/constants/work.js
const workExperience = [
    {
        title: 'Career Pivot — Independent Builder',
        text: `Stepped back from full-time engineering roles to spend a year as an independent builder...`,
        timeLine: '2028 - 2029. Remote',

    },
```

<Tip>
Omit the `image` field for roles without a logo, including transition or "career pivot" entries. The layout adjusts automatically.
</Tip>

## Documenting career pivots

Career pivot entries describe a transition period rather than a specific job. Use them when a meaningful gap or shift in focus belongs on the timeline.

- Prefix the `title` with `Career Pivot —` so visitors can scan the list and immediately tell pivots apart from standard roles.
- Use `text` to explain the motivation, what you shipped or learned during the period, and how it shaped your next move.
- Set `timeLine` to a year range (for example, `2028 - 2029. Remote`) and use `Present` for an ongoing pivot.
- Leave out `image` since pivots aren't tied to a single company.

## When to update this list

Add a new entry whenever you start, finish, or want to highlight a role or career pivot on the personal website. Order entries reverse-chronologically so the most recent entry appears first.
