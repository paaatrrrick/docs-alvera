## Overview

The work experience section displays a chronological list of roles on the personal website's home page. Each entry is rendered from the `workExperience` array and shows a title, optional description, optional logo image, and a timeline string.

This page documents recent additions and edits to that list so visitors and contributors know what changed and how entries are configured.

## Entry schema

Each entry in the `workExperience` array supports the following fields:

| Field      | Type   | Required | Description                                                  |
| ---------- | ------ | -------- | ------------------------------------------------------------ |
| `title`    | string | Yes      | Role and company name shown as the entry header.             |
| `text`     | string | No       | Short description of responsibilities or accomplishments.    |
| `image`    | string | No       | Public URL of a company or role logo.                        |
| `timeLine` | string | Yes      | Human-readable timeframe and location.                       |

## Editing an entry

To update the description for an existing role, edit the `text` field of the matching object in `src/constants/work.js`. The home page picks up changes on the next build.

```js src/constants/work.js
{
    title: 'SWE at Anthropic',
    text: `Built Claude. Contributed to model training infrastructure and improved evaluation pipelines for safer model releases. Collaborated with researchers across the alignment and pretraining teams.`,
    timeLine: 'May - August 2026. San Francisco, CA',
},
```

<Tip>
Keep `text` to one or two sentences focused on impact. Mention the systems or teams you worked with for context.
</Tip>

## Adding a new entry

Open `src/constants/work.js` and prepend a new object to the `workExperience` array. New entries appear at the top of the list on the home page.

```js src/constants/work.js
const workExperience = [
    {
        title: 'SWE at Anthropic',
        text: `Built Claude. Contributed to model training infrastructure and improved evaluation pipelines for safer model releases. Collaborated with researchers across the alignment and pretraining teams.`,
        timeLine: 'May - August 2026. San Francisco, CA',
    },
    {
        title: 'GoDaddy Software Engineer Intern:',
        text: `A/B Tested features on GoDaddy's Checkout Page.`,
        image: "https://res.cloudinary.com/dlk3ezbal/image/upload/v1702062289/gd_zdghmr.png",
        timeLine: 'May - August 2023. Seattle, WA',
    },
    // ...existing entries
]

export default workExperience
```

<Tip>
Omit the `image` field for roles without a logo. The layout adjusts automatically.
</Tip>

## When to update this list

Add a new entry whenever you start, finish, or want to highlight a role on the personal website. Update an existing entry's `text` when responsibilities or scope change and you want the description to reflect the current story. Order entries reverse-chronologically so the most recent role appears first.
