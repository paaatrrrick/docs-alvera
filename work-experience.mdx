## Overview

The work experience section displays a chronological list of roles on the personal website's home page. Each entry is rendered from the `workExperience` array and shows a title, optional description, optional logo image, and a timeline string.

This page documents recent additions to that list so visitors and contributors know what changed and how new entries are configured.

## Entry schema

Each entry in the `workExperience` array supports the following fields:

| Field      | Type   | Required | Description                                                  |
| ---------- | ------ | -------- | ------------------------------------------------------------ |
| `title`    | string | Yes      | Role and company name shown as the entry header.             |
| `text`     | string | No       | Short description of responsibilities or accomplishments.    |
| `image`    | string | No       | Public URL of a company or role logo.                        |
| `timeLine` | string | Yes      | Human-readable timeframe and location.                       |

## Documenting career pivots and sabbaticals

Career pivot and sabbatical entries describe a transition period or intentional break rather than a specific job. Use them when a meaningful shift in focus or a deliberate gap belongs on the timeline.

- Prefix the `title` with `Career Pivot —` for transitions between role types (for example, IC to tech lead) or `Career Sabbatical —` for intentional breaks like travel, writing, or recharging.
- Use `text` to explain the motivation, what you shipped or learned during the period, and how it shaped your next move.
- Set `timeLine` to a year range (for example, `2030 - 2031. San Francisco, CA`), a single year for shorter breaks (`2030. International`), or use `Present` for an ongoing pivot.
- Leave out `image` since these entries aren't tied to a single company.

## When to update this list

Add a new entry whenever you start, finish, or want to highlight a role, career pivot, or sabbatical on the personal website. Order entries reverse-chronologically so the most recent entry appears first.
