Mintlify は、MDX ファイルの Git リポジトリを、高速で検索可能なドキュメントサイトへと変換します。このページでは、内部で何が起きているのかが分かるよう、主要な構成要素を解説します。

## コミットから公開ページまで

1. **コンテンツを書きます。** ページは MDX で執筆します。MDX は Markdown に `<Card>`、`<Tabs>`、`<Accordion>` などの React コンポーネントを加えたものです。
2. **ナビゲーションを設定します。** `docs.json` ではタブ、グループ、テーマ、ナビバーやフッターなどのグローバル要素を定義します。
3. **GitHub にプッシュします。** Mintlify の GitHub アプリがデプロイブランチ上の変更を検出します。
4. **Mintlify がサイトをビルドします。** コンテンツが検証され、リンクがチェックされ、サイトが静的アセットにコンパイルされます。
5. **サイトが公開されます。** 新しいビルドが通常は数秒以内にドメインへデプロイされます。

## ローカル開発

プッシュ前に変更をプレビューするには、Mintlify CLI を使用します:

```bash
mint dev
```

このコマンドは、ホットリロードを有効にしたうえで `http://localhost:3000` でドキュメントを配信します。問題を早期に発見するには、次を実行してください:

```bash
mint validate
mint broken-links
```

## 執筆モデル

- **MDX ファイル** — 各ページは YAML フロントマター（`title`、`description`）を持つ `.mdx` ファイルです。
- **コンポーネント** — Mintlify には、コールアウト、タブ、コードグループ、アコーディオン、カードなどの組み込みコンポーネントが用意されています。
- **スニペット** — 再利用可能なコンテンツは `snippets/` に置き、任意のページにインポートできます。
- **画像とアセット** — リポジトリのルートからの相対パスで参照します。

## 設定

`docs.json` はサイト構造に関する唯一の信頼できる情報源です:

- `navigation` — タブ、グループ、ページの並び順。
- `colors`、`logo`、`favicon` — ブランディング。
- `navbar`、`footer` — グローバルリンクと SNS。
- `contextual` — 各ページの「Copy / Open in ChatGPT」メニューを制御します。

## 検索と AI

公開されたサイトには、いずれも次の機能が含まれます:

- **全文検索** — ビルドのたびに自動的にインデックスされます。
- **AI チャット** — ドキュメントをコンテキストとして質問に回答します。
- **MCP / LLM 連携** — Cursor や Claude などのツールがドキュメントを参照できます。

## デプロイ環境

| 環境 | トリガー | 用途 |
| ---- | -------- | ---- |
| ローカル | `mint dev` | 変更の下書きとプレビュー |

| プロダクション | デプロイブランチへのマージ | 公開サイト |
