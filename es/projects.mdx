## Información general

La página de proyectos destacados se encuentra en `/projects` y muestra una cuadrícula adaptable de proyectos paralelos. Cada tarjeta muestra el nombre del proyecto (enlazado a su repositorio), una insignia de estado Activo o Archivado, una breve descripción corta, una descripción más larga y una lista de etiquetas de tecnologías. Las personas que visitan la página pueden acotar la lista con un cuadro de búsqueda de texto libre y una fila de botones de filtro por tecnología.

La página se basa en datos: cada tarjeta se representa a partir de una entrada del arreglo `projects` exportado por `src/constants/projects.js`. Para añadir, eliminar o actualizar un proyecto, edita ese arreglo: no se requieren cambios en el componente.

## Cuándo usar esta página

- **Añadir un nuevo proyecto paralelo** que quieras destacar a las visitas.
- **Marcar un proyecto como archivado** cuando dejes de mantenerlo pero quieras que siga siendo localizable.
- **Actualizar las etiquetas de tecnologías** para que un proyecto aparezca bajo el botón de filtro correcto.

Si solo quieres ajustar el aspecto de las tarjetas, edita `src/components/Projects.js` y `src/components/projects.css` en su lugar.

## Esquema de entrada

Cada entrada del arreglo `projects` admite los siguientes campos. Todos los campos son obligatorios.

| Campo         | Tipo       | Descripción                                                                                       |
| ------------- | ---------- | ------------------------------------------------------------------------------------------------- |
| `name`        | string     | Nombre del proyecto, mostrado como encabezado enlazado de la tarjeta.                             |
| `tagline`     | string     | Resumen de una línea que se muestra justo debajo del nombre.                                      |
| `description` | string     | Una o dos oraciones de detalle. Se incluye en la búsqueda.                                        |
| `status`      | string     | `'Active'` o `'Archived'`. Determina el color de la insignia.                                     |
| `tech`        | string[]   | Etiquetas de tecnologías que se muestran en la tarjeta y se agregan a la barra de filtros.        |
| `link`        | string     | URL a la que enlaza el encabezado de la tarjeta. Se abre en una nueva pestaña.                    |

### Insignias de estado

El campo `status` representa una pastilla de color junto al nombre del proyecto:

- `'Active'`: proyectos que sigues publicando o manteniendo.
- `'Archived'`: proyectos que son de solo lectura o que ya no se mantienen.

La clase de la insignia se deriva de `status.toLowerCase()`, por lo que el valor debe coincidir exactamente con una de esas dos cadenas para que se aplique el estilo.

### Etiquetas de tecnologías y barra de filtros

Cada cadena única del arreglo `tech` de cualquier proyecto se convierte en un botón en la barra de filtros, ordenado alfabéticamente. Un botón `All` (el predeterminado) muestra todos los proyectos. Seleccionar un filtro de tecnología oculta cualquier tarjeta cuyo arreglo `tech` no incluya esa etiqueta.

Mantén la ortografía de las etiquetas coherente entre entradas: `Node` y `Node.js` generan dos botones de filtro distintos.

## Añadir un nuevo proyecto

Abre `src/constants/projects.js` y añade un objeto al arreglo `projects`. Las nuevas entradas aparecen en el orden en que se declaran.

```js src/constants/projects.js
const projects = [
    {
        name: 'Inkwell',
        tagline: 'A markdown-first note-taking app',
        description: 'A local-first notes app with bidirectional links, a command palette, and end-to-end encrypted sync.',
        status: 'Active',
        tech: ['React', 'Rust', 'SQLite'],
        link: 'https://github.com/example/inkwell',
    },
    // ...more projects
];

export default projects;
```

<Tip>
Cuando archives un proyecto, cambia `status` de `'Active'` a `'Archived'` en lugar de eliminar la entrada: la tarjeta seguirá siendo localizable mediante la búsqueda y los filtros de tecnologías.
</Tip>

## Cómo se combinan la búsqueda y los filtros

La página tiene dos controles independientes que se cruzan:

1. **Cuadro de búsqueda**: compara la consulta recortada y en minúsculas con los campos `name`, `tagline` y `description` de cada proyecto. Un proyecto se conserva si cualquiera de los tres contiene la consulta como subcadena.
2. **Filtro por tecnología**: conserva solo los proyectos cuyo arreglo `tech` incluya la etiqueta activa. `All` desactiva este filtro.

Un proyecto debe pasar **ambos** filtros para mostrarse. Si el resultado combinado está vacío, la cuadrícula muestra un estado vacío que dice "No projects match your search."

### Escribir contenido apto para la búsqueda

Como la búsqueda solo examina `name`, `tagline` y `description`, asegúrate de que esos campos contengan las palabras que es probable que escriba quien visita la página. Por ejemplo, si un proyecto es tu "rastreador de hábitos", incluye la palabra `habit` en el tagline o la descripción aunque sea obvio por el nombre.

## Cómo escribir buenas entradas

- **`tagline`**: una frase corta. Empieza por la categoría ("Self-hosted bookmark manager") en lugar de un lenguaje de marketing.
- **`description`**: una o dos oraciones. Menciona la mecánica principal y las capacidades destacables (una API, importadores, cifrado).
- **`tech`**: enumera las tecnologías que le interesarían a quien lee, no todas las dependencias. De tres a cinco etiquetas por proyecto es una buena referencia.
- **`link`**: apunta al hogar canónico del proyecto. Un repositorio de GitHub está bien; un sitio alojado es mejor si existe.
