## Empieza en tres pasos

Esta guía rápida te muestra todo lo necesario para publicar un sitio de Mintlify: clonar el repositorio de documentación, previsualizarlo en local con la CLI de Mintlify, hacer tu primera edición de contenido y publicar el cambio. Al terminar, tendrás un flujo de desarrollo local funcional y un sitio publicado que podrás compartir.

### Paso 1: Configura tu entorno local

<AccordionGroup>
  <Accordion icon="copy" title="Clona tu documentación localmente">
    Durante el proceso de incorporación, se creó un repositorio de GitHub con el contenido de tu documentación si no tenías uno previamente. Puedes encontrar el enlace a este repositorio en tu [panel](https://dashboard.mintlify.com).
    
    Para clonar el repositorio localmente y poder hacer y previsualizar cambios en tu documentación, sigue la guía [Cloning a repository](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository) en la documentación de GitHub.
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Inicia el servidor de previsualización">
    1. Instala el CLI de Mintlify: `npm i -g mint`
    2. Navega al directorio de tu documentación y ejecuta: `mint dev`
    3. Abre `http://localhost:3000` para ver tu documentación en vivo.
    
    <Tip>La previsualización se actualiza automáticamente a medida que editas archivos.</Tip>
  </Accordion>
</AccordionGroup>

### Paso 2: Despliega tus cambios

<AccordionGroup>
  <Accordion icon="github" title="Instala nuestra aplicación de GitHub">
    Instala la aplicación de GitHub de Mintlify desde tu [panel](https://dashboard.mintlify.com/settings/organization/github-app).
    
    Nuestra aplicación de GitHub despliega tus cambios automáticamente en tu sitio de documentación, así no tienes que gestionar los despliegues tú mismo.
</Accordion>
<Accordion icon="palette" title="Actualiza el nombre y los colores de tu sitio">
    Como primer cambio, vamos a actualizar el nombre y los colores de tu sitio de documentación.

    1. Abre `docs.json` en tu editor.
    2. Cambia el campo `"name"` por el nombre de tu proyecto.
    3. Actualiza los `"colors"` para que coincidan con tu marca.
    4. Guarda y mira tus cambios al instante en `http://localhost:3000`.

    <Tip>Prueba a cambiar el color primario para ver una diferencia inmediata.</Tip>
  </Accordion>
</AccordionGroup>

### Paso 3: Publica

<Accordion icon="rocket" title="Publica tu documentación">
  1. Confirma (commit) y haz push de tus cambios.
  2. Tu documentación se actualizará y estará en vivo en cuestión de segundos.
</Accordion>
