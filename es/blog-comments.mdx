## Resumen

El widget Comments renderiza una sección de comentarios debajo de una entrada del blog. Carga los comentarios existentes al montar la entrada, permite a los lectores enviar un nuevo comentario mediante un textarea y añade de forma optimista los nuevos comentarios a la lista una vez que se guardan.

El widget es solo de presentación: no se comunica directamente con un backend. Tú le pasas los handlers `fetchComments` y `postComment`, de modo que el mismo componente puede conectarse a cualquier capa de almacenamiento (una API alojada, una función serverless, un archivo JSON estático, etc.).

## Cuándo usarlo

Añade el widget a cualquier entrada del blog en la que quieras que los lectores dejen respuestas cortas de texto. Encaja bien cuando quieres:

- Un hilo de discusión ligero en una sola entrada.
- Control total sobre dónde se almacenan y moderan los comentarios.
- UI optimista, para que un comentario publicado aparezca al instante cuando el handler se resuelve.

Las reacciones con emojis son una función aparte y no forman parte de este widget.

## Props

| Prop            | Tipo                                          | Requerido | Descripción                                                                                                |
| --------------- | --------------------------------------------- | --------- | ---------------------------------------------------------------------------------------------------------- |
| `postId`        | string                                        | Sí        | Identificador de la entrada. Se usa como clave para obtener y publicar comentarios.                        |
| `fetchComments` | `(postId) => Promise<Comment[]>`              | Sí        | Se llama al montar y siempre que cambia `postId`. Devuelve la lista actual de comentarios de la entrada.    |
| `postComment`   | `(postId, body) => Promise<Comment>`          | Sí        | Se llama cuando el lector envía el formulario. Devuelve el comentario recién creado, que se añade a la lista. |

### Forma del comentario

Cada comentario devuelto por `fetchComments` o `postComment` debe incluir:

| Campo    | Tipo   | Descripción                                       |
| -------- | ------ | ------------------------------------------------- |
| `id`     | string | Clave única estable usada para renderizar la lista. |
| `author` | string | Nombre mostrado antes del cuerpo del comentario.  |
| `body`   | string | El texto del comentario.                          |

## Comportamiento

- Al montar y siempre que cambia `postId`, el widget llama a `fetchComments(postId)` y reemplaza la lista visible con el resultado.
- Una petición en curso que finaliza después de que el componente se desmonte (o después de que `postId` cambie de nuevo) se descarta: el estado solo se actualiza para la petición activa más reciente.
- Al enviar el formulario se recorta el borrador, se llama a `postComment(postId, body)` y se añade el comentario devuelto a la lista. El textarea se vacía si la operación tiene éxito.
- Los borradores vacíos o que solo contengan espacios se ignoran y no disparan `postComment`.
- El encabezado muestra el recuento actual como `N comments`.

## Uso

Monta el widget al final del layout de tu entrada de blog y pásale tus propios handlers.

```jsx
import Comments from "./components/comments/Comments";

async function fetchComments(postId) {
  const res = await fetch(`/api/posts/${postId}/comments`);
  return res.json();
}

async function postComment(postId, body) {
  const res = await fetch(`/api/posts/${postId}/comments`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ body }),
  });
  return res.json();
}

export default function BlogPost({ post }) {
  return (
    <article>
      {/* ...cuerpo de la entrada... */}
      <Comments
        postId={post.id}
        fetchComments={fetchComments}
        postComment={postComment}
      />
    </article>
  );
}
```

<Tip>
Devuelve `postComment` con la misma forma que tu backend almacena para que la entrada optimista coincida con la siguiente respuesta de `fetchComments`. Como mínimo, incluye `id`, `author` y `body`.
</Tip>

## Estilos

El widget incluye un pequeño archivo CSS que añade un borde superior sobre la sección, elimina las viñetas de la lista, separa los comentarios con un divisor fino y estira el textarea al ancho del contenedor con una altura mínima. Sobrescribe estos estilos apuntando a `.comments`, `.comments ul`, `.comments li` y `.comments textarea` en tu propia hoja de estilos.

## Insignia con el recuento de comentarios

Usa el componente `CommentCount` para mostrar el total de comentarios de una entrada fuera del widget principal — por ejemplo, junto al título de una entrada en una vista de lista o en una tarjeta del índice del blog.

Lee desde la misma fuente que el widget mediante el hook `useComments`, por lo que el recuento permanece sincronizado con los comentarios mostrados por `Comments`.

### Props

| Prop     | Tipo   | Requerido | Descripción                                                                       |
| -------- | ------ | --------- | --------------------------------------------------------------------------------- |
| `postId` | string | Sí        | Identificador de la entrada cuyos comentarios deben contarse. Se pasa a `useComments`. |

### Comportamiento

- Mientras la petición subyacente está en curso, la insignia no renderiza nada: no hay placeholder ni skeleton.
- Una vez que la petición se resuelve, la insignia muestra `No comments` cuando la lista está vacía, `1 comment` para un único comentario y `N comments` en los demás casos.
- El elemento renderizado es un `<span>` con la clase `comment-count`, por lo que puedes colocarlo en línea junto a otro texto.

### Uso

```jsx
import CommentCount from "./components/comments/CommentCount";

export default function BlogPostCard({ post }) {
  return (
    <a href={`/blog/${post.slug}`}>
      <h2>{post.title}</h2>
      <CommentCount postId={post.id} />
    </a>
  );
}
```

<Tip>
Cada `CommentCount` monta su propia suscripción a `useComments`, por lo que renderizar muchos de ellos en una vista de lista larga lanza una petición por entrada. Si eso se convierte en un problema, añade caché o batching en tu backend.
</Tip>

## Ordenar comentarios

Usa el hook `useCommentSort` junto con el control `CommentSort` para que los lectores puedan reordenar el hilo por más recientes, más antiguos o destacados.

### `useCommentSort(comments)`

Envuelve la lista devuelta por `useComments` (o cualquier handler) y renderiza el resultado ordenado.

| Valor devuelto | Tipo                                    | Descripción                                                                  |
| -------------- | --------------------------------------- | ---------------------------------------------------------------------------- |
| `sort`         | `"newest" \| "oldest" \| "top"`         | El orden seleccionado actualmente. Por defecto es `"newest"`.                |
| `setSort`      | `(value) => void`                       | Actualiza el orden activo.                                                   |
| `sorted`       | `Comment[]`                             | La lista de entrada ordenada por el comparador activo. Memoizada.            |

Comparadores:

- `newest` y `oldest` ordenan por `comment.date` (analizado con `new Date`).
- `top` ordena por la suma de los valores en `comment.reactions`. Los comentarios sin objeto `reactions` se tratan como cero.

### `CommentSort`

Un selector de tres botones que alterna entre los modos de orden.

| Prop       | Tipo                            | Requerido | Descripción                                                              |
| ---------- | ------------------------------- | --------- | ------------------------------------------------------------------------ |
| `value`    | `"newest" \| "oldest" \| "top"` | Sí        | El orden activo. El botón correspondiente recibe la clase `is-active`.   |
| `onChange` | `(value) => void`               | Sí        | Se llama cuando el lector pulsa un botón. Conéctalo a `setSort`.         |

### Uso

```jsx
import Comments from "./components/comments/Comments";
import CommentSort from "./components/comments/CommentSort";
import { useComments } from "./components/comments/useComments";
import { useCommentSort } from "./components/comments/useCommentSort";

export default function BlogPost({ post }) {
  const { comments, addComment } = useComments(post.id);
  const { sort, setSort, sorted } = useCommentSort(comments);

  return (
    <article>
      {/* ...cuerpo de la entrada... */}
      <CommentSort value={sort} onChange={setSort} />
      <Comments
        postId={post.id}
        fetchComments={() => Promise.resolve(sorted)}
        postComment={(_, body) => addComment(body)}
      />
    </article>
  );
}
```

<Tip>
Incluye un campo `date` en cada comentario para que `newest` y `oldest` ordenen de forma determinista. Los comentarios sin `date` recaen en una fecha inválida, lo que deja indefinido su orden relativo.
</Tip>
