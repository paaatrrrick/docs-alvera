Este glosario define los términos que encontrarás a lo largo de la documentación.

## MDX

Un formato de archivo que combina Markdown con JSX, lo que te permite incrustar componentes interactivos directamente dentro del contenido. Todas las páginas de esta documentación se escriben como archivos `.mdx`.

## Frontmatter

El bloque YAML al inicio de cada archivo MDX, delimitado por `---`. Define los metadatos a nivel de página, como `title`, `description` e `icon`, que Mintlify usa para renderizar la página.

## Navegación

La estructura definida en `docs.json` que organiza las páginas en pestañas, grupos e idiomas. La navegación controla la barra lateral, el encabezado y el enrutamiento de tu sitio de documentación.

## Snippet

Un fragmento reutilizable de contenido MDX almacenado en el directorio `snippets/`. Importa un snippet en cualquier página para compartir contenido en la documentación sin duplicarlo. Consulta [Snippets reutilizables](/essentials/reusable-snippets).

## Referencia de API

Documentación generada automáticamente para los endpoints de tu API a partir de una especificación OpenAPI. Cada endpoint se convierte en una página interactiva donde los usuarios pueden inspeccionar parámetros y probar solicitudes. Consulta [Documentación de API](/api-reference/introduction).
