---
title: 'Configuración global en docs.json'
description: 'Referencia de todas las propiedades de docs.json que controlan el nombre del sitio, el tema, los colores, el logotipo, la navegación y las integraciones de tu documentación de Mintlify.'
icon: 'gear'
---

Todo sitio de Mintlify necesita un archivo `docs.json` con la configuración principal. Aprende más sobre las [propiedades](#properties) a continuación.

## Propiedades

<ResponseField name="name" type="string" required>
Nombre de tu proyecto. Se usa para el título global.

Ejemplo: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array de grupos con todas las páginas dentro de cada grupo
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    El nombre del grupo.

    Ejemplo: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Las rutas relativas a los archivos markdown que servirán como páginas.

    Ejemplo: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ruta a la imagen del logotipo u objeto con la ruta de los logotipos para los modos "light" y "dark"
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Ruta al logotipo en modo claro
    </ResponseField>
    <ResponseField name="dark" type="string">
      Ruta al logotipo en modo oscuro
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      Adónde te lleva al hacer clic en el logotipo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ruta a la imagen del favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de color hexadecimales para tu tema global
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      El color principal. Se usa con más frecuencia para resaltar contenido,
      encabezados de sección y acentos en modo claro
    </ResponseField>
    <ResponseField name="light" type="string">
      El color principal para el modo oscuro. Se usa con más frecuencia para resaltar
      contenido, encabezados de sección y acentos en modo oscuro
    </ResponseField>
    <ResponseField name="dark" type="string">
      El color principal para botones importantes
    </ResponseField>
    <ResponseField name="background" type="object">
      El color de fondo en modo claro y oscuro
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          El código de color hexadecimal del fondo en modo claro
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          El código de color hexadecimal del fondo en modo oscuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de `name`s y `url`s de los enlaces que quieres incluir en la barra superior
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    El nombre del botón.

    Ejemplo: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    La URL a la que lleva al hacer clic en el botón. Ejemplo: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link muestra un botón. GitHub muestra la información del repositorio en la URL proporcionada, incluido el número de estrellas de GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Si es `link`: a qué enlaza el botón.
    
    Si es `github`: enlace al repositorio del que cargar la información de GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Texto dentro del botón. Solo es obligatorio si `type` es `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nombres de versiones. Úsalo solo si quieres mostrar diferentes versiones
  de la documentación con un desplegable en la barra de navegación.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un array de anchors, que incluye el `icon`, el `color` y la `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    El icono de [Font Awesome](https://fontawesome.com/search?q=heart) usado para destacar el anchor.

    Ejemplo: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    El nombre de la etiqueta del anchor.

    Ejemplo: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      El inicio de la URL que marca qué páginas pertenecen al anchor. Generalmente es el nombre de la carpeta donde colocas tus páginas.
    </ResponseField>
    <ResponseField name="color" type="string">
      El color hexadecimal del fondo del icono del anchor. También puede ser un degradado si pasas un objeto con las propiedades `from` y `to`, cada una con un color hexadecimal.
    </ResponseField>
    <ResponseField name="version" type="string">
      Se usa si quieres ocultar un anchor hasta que se seleccione la versión correcta de la documentación.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Pasa `true` si quieres ocultar el anchor hasta que enlaces directamente a alguien con la documentación que contiene.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: "brands", "duotone", "light", "sharp-solid", "solid" o "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Sobrescribe las configuraciones por defecto del anchor superior.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      El nombre del anchor superior
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Icono de Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: "brands", "duotone", "light", "sharp-solid", "solid" o "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un array de pestañas de navegación.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      El nombre de la etiqueta de la pestaña.
    </ResponseField>
    <ResponseField name="url" type="string">
      El inicio de la URL que marca qué páginas pertenecen a la pestaña. Generalmente
      es el nombre de la carpeta donde colocas tus páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuración de los ajustes de la API. Aprende más sobre las páginas de API en [API Components](https://mintlify.com/docs/api-playground/overview).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      La URL base para todos los endpoints de la API. Si `baseUrl` es un array, se habilitarán múltiples
      opciones de URL base que el usuario podrá alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          La estrategia de autenticación utilizada para todos los endpoints de la API.
        </ResponseField>
        <ResponseField name="name" type="string">
        El nombre del parámetro de autenticación utilizado en el playground de la API.

        Si el método es `basic`, el formato debe ser `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        El valor por defecto diseñado para servir de prefijo en el campo de entrada de autenticación.

        Por ejemplo, un `inputPrefix` de `AuthKey` heredará el resultado de entrada por defecto del campo de autenticación como `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuraciones para el playground de la API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Indica si el playground se muestra, se oculta o solo muestra el endpoint sin interactividad adicional del usuario (`simple`).

          Aprende más en las [guías del playground](https://mintlify.com/docs/api-playground/overview)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Habilitar este indicador garantiza que el orden de las claves en las páginas de OpenAPI coincida con el orden definido en el archivo OpenAPI.

      <Warning>Pronto este comportamiento estará habilitado por defecto, momento en el que este campo quedará obsoleto.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una cadena o un array de cadenas con la(s) URL o ruta(s) relativa(s) que apuntan a tu
  archivo OpenAPI.
  
  Ejemplos:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objeto de cuentas de redes sociales donde el par clave:propiedad representa la plataforma social y la URL de la cuenta.
  
  Ejemplo: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Uno de los siguientes valores: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Ejemplo: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    La URL a la plataforma social.
    
    Ejemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configuraciones para habilitar los botones de feedback

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Habilita un botón que permite a los usuarios sugerir ediciones mediante pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Habilita un botón que permite a los usuarios reportar un problema sobre la documentación
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personaliza el conmutador del modo oscuro.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Defínelo si siempre quieres mostrar el modo claro u oscuro a los nuevos usuarios. Cuando no
      se establece, usamos por defecto el mismo modo que el sistema operativo del usuario.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Establécelo en true para ocultar el conmutador de modo claro/oscuro. Puedes combinar `isHidden` con `default` para forzar a tu documentación a usar únicamente el modo claro u oscuro. Por ejemplo:
      
      <CodeGroup>
      ```json Only Dark Mode
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Only Light Mode
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Una imagen de fondo que se mostrará detrás de cada página. Mira un ejemplo con
  [Infisical](https://infisical.com/docs) y [FRPC](https://frpc.io).
</ResponseField>
