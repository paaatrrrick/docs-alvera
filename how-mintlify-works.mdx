## From commit to published page

1. **You write content.** Pages are authored in MDX, which is Markdown plus React components like `<Card>`, `<Tabs>`, and `<Accordion>`.
2. **You configure navigation.** `docs.json` defines tabs, groups, theming, and global elements like the navbar and footer.
3. **You push to GitHub.** Mintlify's GitHub app detects the change on your deploy branch.
4. **Mintlify builds the site.** Content is validated, links are checked, and the site is compiled into static assets.
5. **The site goes live.** The new build is deployed to your domain, usually within seconds.

## Local development

Use the Mintlify CLI to preview changes before pushing:

```bash
mint dev
```

This serves your docs at `http://localhost:3000` with hot reload. To catch issues early, run:

```bash
mint validate
mint broken-links
```

## Authoring model

- **MDX files** — every page is an `.mdx` file with YAML frontmatter (`title`, `description`).
- **Components** — Mintlify ships built-in components for callouts, tabs, code groups, accordions, cards, and more.
- **Snippets** — reusable content lives in `snippets/` and can be imported into any page.
- **Images and assets** — referenced by relative path from the repo root.

## Configuration

`docs.json` is the single source of truth for site structure:

- `navigation` — tabs, groups, and page ordering.
- `colors`, `logo`, `favicon` — branding.
- `navbar`, `footer` — global links and socials.
- `contextual` — controls the "Copy / Open in ChatGPT" menu on each page.

## Search and AI

Every published site includes:

- **Full-text search** indexed automatically on each build.
- **AI chat** that answers questions using your docs as context.
- **MCP and LLM integrations** so tools like Cursor and Claude can reference your docs.
