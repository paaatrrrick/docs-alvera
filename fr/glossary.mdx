Ce glossaire définit les termes que vous rencontrerez dans cette documentation. Parcourez-le une fois, puis revenez-y dès qu'un mot n'est pas clair.

## Termes

- **Déploiement (Deployment)** — Une version publiée de votre site. Chaque push sur votre branche principale crée un nouveau déploiement.
- **Prévisualisation (Preview)** — Une compilation temporaire d'une pull request qui vous permet de relire les modifications avant leur mise en ligne.
- **Navigation** — La structure de la barre latérale qui organise vos pages, définie dans votre fichier de configuration `docs.json`.
- **Extrait (Snippet)** — Un bloc de contenu réutilisable stocké dans `snippets/` que vous pouvez inclure sur plusieurs pages.
- **Frontmatter** — Le bloc YAML en haut de chaque fichier MDX qui définit les métadonnées de la page, comme `title` et `description`.

Si vous débutez, lisez d'abord le [guide de démarrage rapide](/quickstart), puis configurez votre environnement avec le [guide de développement local](/development).
