## Vue d'ensemble

La section expérience professionnelle affiche une liste chronologique des postes sur la page d'accueil du site personnel. Chaque entrée est rendue à partir du tableau `workExperience` et affiche un titre, une description optionnelle, une image de logo optionnelle et une chaîne de calendrier.

Cette page documente les ajouts récents à cette liste afin que les visiteurs et les contributeurs sachent ce qui a changé et comment les nouvelles entrées sont configurées.

## Schéma d'une entrée

Chaque entrée du tableau `workExperience` prend en charge les champs suivants :

| Champ      | Type   | Requis | Description                                                  |
| ---------- | ------ | -------- | ------------------------------------------------------------ |
| `title`    | string | Oui      | Poste et nom de l'entreprise affichés en en-tête d'entrée.   |
| `text`     | string | Non      | Brève description des responsabilités ou des réalisations.   |
| `image`    | string | Non      | URL publique d'un logo de l'entreprise ou du poste.          |
| `timeLine` | string | Oui      | Période et lieu lisibles par un humain.                      |

## Rédiger les descriptions de poste

Utilisez le champ `text` pour communiquer l'impact, pas seulement les responsabilités.

- Commencez par la contribution à plus fort impact.
- Mentionnez des domaines de focus concrets (par exemple, fiabilité, latence, pipelines d'évaluation).
- Mettez en avant la collaboration inter-équipes lorsqu'elle apporte du contexte au poste.
- Limitez la description à deux ou trois phrases pour que la page d'accueil reste rapide à parcourir.

## Quand mettre à jour cette liste

Ajoutez une nouvelle entrée chaque fois que vous commencez, terminez ou souhaitez mettre en avant un poste sur le site personnel. Classez les entrées dans l'ordre chronologique inverse pour que le poste le plus récent apparaisse en premier.
