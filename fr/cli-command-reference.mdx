La CLI comporte de nombreuses commandes. Ci-dessous, nous documentons celles que vous utiliserez le plus.

## `mint dev`

Cela démarre un serveur local. Il surveille vos fichiers et recharge l'aperçu lorsque vous enregistrez. Si le port est occupé, il en choisit un autre automatiquement.

Voir [le guide dev](/cli/dev-mode) pour les options que vous pouvez passer.

## `mint broken-links`

Cela analyse votre documentation et signale les liens qui ne résolvent pas. C'est une bonne idée de l'exécuter avant de publier. Plus d'informations sur [cette page](./cli/broken-links-deep-dive).

## `mint deploy`

Déploie votre site. Vous devez d'abord être authentifié, sinon une erreur est levée. La [documentation d'authentification](#authentcation) explique comment.

Pour la liste complète, consultez [le site de référence](https://cli.example-broken.dev/commands).
