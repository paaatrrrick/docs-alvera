---

## Aperçu

Le compteur de vues affiche combien de fois un article de blog a été lu. Il se compose de deux pièces :

- `useViewCount` : un hook React qui dialogue avec le service de vues pour enregistrer une vue ou lire le total actuel d'un article.
- `ViewCounter` : un petit composant qui affiche le nombre formaté à côté d'un article (par exemple, `1 204 vues`).

Utilisez-le sur toute page qui représente un article unique et possède un `postId` stable.

## Quand l'utiliser

- Ajoutez `ViewCounter` à la mise en page d'un article quand vous voulez montrer aux personnes qui lisent à quel point un article est populaire.
- Appelez `useViewCount` directement quand vous avez besoin du nombre brut pour autre chose que le libellé par défaut — par exemple, trier une liste d'articles par popularité ou masquer un badge « tendance » tant qu'un seuil n'est pas franchi.

Le hook enregistre une vue par défaut, alors montez-le une seule fois par rendu d'article. Si vous avez besoin de lire le compteur sans enregistrer une vue (par exemple, dans une vue en liste), passez `increment: false`.

## Composant ViewCounter

`ViewCounter` prend une seule prop `postId` et affiche un `<span>` contenant le nombre formaté. Pendant que la requête est en cours, ou si le service de vues est injoignable, le composant n'affiche rien afin de ne jamais montrer un zéro trompeur.

```jsx
import ViewCounter from "./components/ViewCounter";

export default function PostHeader({ post }) {
  return (
    <header>
      <h1>{post.title}</h1>
      <p>
        {post.publishedAt} · <ViewCounter postId={post.slug} />
      </p>
    </header>
  );
}
```

### Props

| Prop     | Type   | Requis | Description                                                                              |
| -------- | ------ | ------ | ---------------------------------------------------------------------------------------- |
| `postId` | string | Oui    | Identifiant stable de l'article. Le slug est un bon choix — il ne doit pas changer.       |

## Hook useViewCount

`useViewCount(postId, options)` renvoie le dernier nombre de vues pour un article, ou `null` pendant que la requête est en attente ou si elle échoue.

```jsx
import { useViewCount } from "./hooks/useViewCount";

function TrendingBadge({ postId }) {
  const count = useViewCount(postId, { increment: false });
  if (count === null || count < 1000) return null;
  return <span className="badge">🔥 Tendance</span>;
}
```

### Paramètres

| Paramètre           | Type    | Valeur par défaut | Description                                                                                  |
| ------------------- | ------- | ----------------- | -------------------------------------------------------------------------------------------- |
| `postId`            | string  | —                 | Identifiant stable de l'article.                                                             |
| `options.increment` | boolean | `true`            | Quand vaut `true`, envoie un `POST` et enregistre une nouvelle vue. Quand vaut `false`, envoie un `GET` en lecture seule. |

### Valeur de retour

Le hook renvoie le nombre de vues actuel sous forme de nombre, ou `null` quand le nombre n'est pas encore disponible ou que la requête a échoué. Traitez `null` comme « inconnu » : affichez un repli, masquez l'interface ou attendez.

<Tip>
Montez `ViewCounter` (ou appelez `useViewCount` avec la valeur par défaut `increment: true`) exactement une fois par affichage d'article. Le monter plusieurs fois sur la même page comptabilise la lecture en double.
</Tip>

## Combiner avec la signature de l'article

Pour la plupart des mises en page d'article, vous voulez afficher le nombre de vues à côté de la date de publication et du temps de lecture. Le composant `PostMeta` réunit les trois dans une seule signature afin que l'en-tête de chaque article reste cohérent.

```jsx
import PostMeta from "./components/PostMeta";

export default function PostHeader({ post }) {
  return (
    <header>
      <h1>{post.title}</h1>
      <PostMeta post={post} />
    </header>
  );
}
```

`PostMeta` lit les champs suivants sur le `post` qu'il reçoit :

| Champ  | Type     | Utilisé pour                                                                                  |
| ------ | -------- | --------------------------------------------------------------------------------------------- |
| `date` | string   | Affiché tel quel quand il est présent. Passez-le préformaté (par exemple, `« 12 mai 2026 »`). |
| `body` | string[] | Joint avec des espaces et passé à `ReadingTime` pour estimer la durée de lecture.             |
| `id`   | string   | Passé à `ViewCounter` comme `postId`. Doit être stable pour que le compteur soit fiable.       |

Comme `PostMeta` monte `ViewCounter` en interne, n'affichez pas non plus `ViewCounter` séparément sur la même page — cela compterait la lecture en double. Utilisez `PostMeta` pour la signature, et ne recourez au `ViewCounter` autonome ou à `useViewCount` que sur les pages qui n'affichent pas déjà une signature (par exemple, des pages de liste ou des badges de tendance).

## Total des vues du site

`SiteViews` et `useTotalViews` affichent le nombre cumulé de lectures sur l'ensemble des articles. Utilisez-les sur la page d'accueil, le pied de page ou la page « à propos » quand vous voulez un seul chiffre global plutôt qu'un compteur par article.

```jsx
import SiteViews from "./components/SiteViews";

export default function Footer() {
  return (
    <footer>
      <SiteViews />
    </footer>
  );
}
```

`SiteViews` ne prend aucune prop et affiche un `<p>` du type `12 540 lectures au total sur l'ensemble des articles.` Tant que la requête n'est pas résolue, ou si elle échoue, le composant n'affiche rien.

Pour une interface personnalisée, appelez `useTotalViews()` directement. Il renvoie le total actuel sous forme de nombre, ou `null` pendant l'attente ou en cas d'échec.

```jsx
import { useTotalViews } from "./hooks/useTotalViews";

function MilestoneBanner() {
  const total = useTotalViews();
  if (total === null || total < 10000) return null;
  return <div className="banner">🎉 Plus de {total.toLocaleString()} lectures !</div>;
}
```

Contrairement à `useViewCount`, `useTotalViews` ne fait que lire — il n'incrémente jamais rien.

## Service de vues

Le composant et les hooks appellent le service de vues à `https://api.paaatrrrick.com/views` :

- `POST /views/:postId` : incrémente le compteur pour `postId` et renvoie le nouveau total sous la forme `{ "count": <nombre> }`.
- `GET /views/:postId` : renvoie le total actuel d'un seul article sous la forme `{ "count": <nombre> }` sans l'incrémenter.
- `GET /views/total` : renvoie le total cumulé sur l'ensemble des articles sous la forme `{ "total": <nombre> }`.

Si le service répond avec un statut différent d'OK ou si la réponse ne peut pas être analysée, le compteur reste à `null` et l'interface n'affiche rien. Le composant ne réessaie pas — actualisez la page pour réessayer.
