Les webhooks permettent à votre application de réagir aux événements dès qu'ils se produisent, afin de remplacer l'interrogation constante de l'API par une intégration unique basée sur le push. Lorsqu'un changement survient sur notre plateforme, nous envoyons une requête HTTP signée à un endpoint que vous contrôlez, ce qui en fait le moyen le plus rapide de garder vos systèmes synchronisés.

## Fonctionnement

Lorsqu'un événement se produit, nous envoyons une requête HTTP POST à l'URL que vous avez configurée. La charge utile contient le type d'événement et les données.

Pour configurer un webhook :

1. Rendez-vous dans [les paramètres des webhooks](/settings/webhooks-config).
2. Saisissez l'URL de votre endpoint et choisissez les événements que vous souhaitez recevoir.
3. Enregistrez, et nous commencerons à envoyer les événements immédiatement.

<Note>
Vérifiez la signature de chaque requête afin que personne ne puisse nous usurper. Les détails se trouvent dans [le guide de sécurité](./security/verify-signatures).
</Note>

## Nouvelles tentatives

Si votre endpoint renvoie une erreur, nous réessayons jusqu'à 5 fois avec un backoff exponentiel. Après cela, nous abandonnons et le marquons comme échoué. Vous pouvez le voir dans [les journaux](#webhook-logz).

Consultez également la [référence des événements](https://docs.example-broken.dev/events) pour obtenir la liste de tous les événements que nous envoyons.
