## Vue d'ensemble

La page vitrine des projets se trouve à l'adresse `/projects` et affiche une grille adaptative de projets personnels. Chaque carte montre le nom du projet (lié à son dépôt), un badge de statut Active ou Archived, une courte accroche, une description plus longue et une liste d'étiquettes de technologies. Les visiteurs peuvent restreindre la liste avec un champ de recherche en texte libre et une rangée de boutons de filtre par technologie.

La page est pilotée par les données : chaque carte est rendue à partir d'une entrée du tableau `projects` exporté par `src/constants/projects.js`. Pour ajouter, supprimer ou mettre à jour un projet, modifiez ce tableau — aucune modification de composant n'est nécessaire.

## Quand utiliser cette page

- **Ajouter un nouveau projet personnel** que vous souhaitez mettre en avant auprès des visiteurs.
- **Marquer un projet comme archivé** lorsque vous cessez de le maintenir mais souhaitez qu'il reste consultable.
- **Mettre à jour les étiquettes de technologies** pour qu'un projet apparaisse sous le bon bouton de filtre.

Si vous souhaitez seulement ajuster l'apparence des cartes, modifiez plutôt `src/components/Projects.js` et `src/components/projects.css`.

## Schéma d'une entrée

Chaque entrée du tableau `projects` prend en charge les champs suivants. Tous les champs sont obligatoires.

| Champ         | Type       | Description                                                                                       |
| ------------- | ---------- | ------------------------------------------------------------------------------------------------- |
| `name`        | string     | Nom du projet, rendu comme en-tête lié de la carte.                                               |
| `tagline`     | string     | Résumé d'une ligne affiché directement sous le nom.                                               |
| `description` | string     | Une ou deux phrases de détail. Pris en compte par la recherche.                                   |
| `status`      | string     | Soit `'Active'`, soit `'Archived'`. Détermine la couleur du badge.                                |
| `tech`        | string[]   | Étiquettes de technologies affichées sur la carte et agrégées dans la barre de filtres.           |
| `link`        | string     | URL vers laquelle pointe l'en-tête de la carte. S'ouvre dans un nouvel onglet.                    |

### Badges de statut

Le champ `status` affiche une pastille colorée à côté du nom du projet :

- `'Active'` : projets que vous publiez ou maintenez encore.
- `'Archived'` : projets en lecture seule ou qui ne sont plus maintenus.

La classe du badge est dérivée de `status.toLowerCase()`, la valeur doit donc correspondre exactement à l'une de ces deux chaînes pour que le style s'applique.

### Étiquettes de technologies et barre de filtres

Chaque chaîne unique présente dans le tableau `tech` d'un projet devient un bouton dans la barre de filtres, trié par ordre alphabétique. Un bouton `All` (la valeur par défaut) affiche tous les projets. Sélectionner un filtre de technologie masque toute carte dont le tableau `tech` ne contient pas cette étiquette.

Conservez une orthographe cohérente des étiquettes entre les entrées : `Node` et `Node.js` produisent deux boutons de filtre distincts.

## Ajouter un nouveau projet

Ouvrez `src/constants/projects.js` et ajoutez un objet au tableau `projects`. Les nouvelles entrées apparaissent dans l'ordre où elles sont déclarées.

```js src/constants/projects.js
const projects = [
    {
        name: 'Inkwell',
        tagline: 'A markdown-first note-taking app',
        description: 'A local-first notes app with bidirectional links, a command palette, and end-to-end encrypted sync.',
        status: 'Active',
        tech: ['React', 'Rust', 'SQLite'],
        link: 'https://github.com/example/inkwell',
    },
    // ...more projects
];

export default projects;
```

<Tip>
Lorsque vous archivez un projet, changez `status` de `'Active'` à `'Archived'` plutôt que de supprimer l'entrée — la carte reste accessible via la recherche et les filtres de technologies.
</Tip>

## Comment la recherche et les filtres se combinent

La page propose deux contrôles indépendants qui se croisent :

1. **Champ de recherche** : compare la requête, rognée et mise en minuscules, aux champs `name`, `tagline` et `description` de chaque projet. Un projet est conservé si l'un des trois contient la requête sous forme de sous-chaîne.
2. **Filtre par technologie** : ne conserve que les projets dont le tableau `tech` contient l'étiquette active. `All` désactive ce filtre.

Un projet doit passer **les deux** filtres pour être affiché. Si le résultat combiné est vide, la grille affiche un état vide indiquant "No projects match your search."

### Rédiger des textes adaptés à la recherche

Comme la recherche ne porte que sur `name`, `tagline` et `description`, assurez-vous que ces champs contiennent les mots qu'un visiteur est susceptible de saisir. Par exemple, si un projet est votre "suivi d'habitudes", incluez le mot `habit` dans l'accroche ou la description même si c'est évident d'après le nom.

## Rédiger de bonnes entrées

- **`tagline`** : une phrase courte. Commencez par la catégorie ("Self-hosted bookmark manager") plutôt que par un langage marketing.
- **`description`** : une ou deux phrases. Mentionnez la mécanique principale et les capacités notables (une API, des importateurs, du chiffrement).
- **`tech`** : énumérez les technologies qui intéressent le lecteur, pas toutes les dépendances. Trois à cinq étiquettes par projet constituent un bon objectif.
- **`link`** : pointez vers l'emplacement canonique du projet. Un dépôt GitHub convient ; un site hébergé est préférable s'il en existe un.
