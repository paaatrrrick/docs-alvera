Ce glossaire définit les termes que vous rencontrerez tout au long de la documentation.

## MDX

Un format de fichier qui combine Markdown et JSX, vous permettant d'intégrer des composants interactifs directement dans votre contenu. Toutes les pages de cette documentation sont rédigées sous forme de fichiers `.mdx`.

## Frontmatter

Le bloc YAML en haut de chaque fichier MDX, délimité par `---`. Il définit les métadonnées de la page, telles que `title`, `description` et `icon`, que Mintlify utilise pour afficher la page.

## Navigation

La structure définie dans `docs.json` qui organise les pages en onglets, groupes et langues. La navigation contrôle la barre latérale, l'en-tête et le routage de votre site de documentation.

## Snippet

Un fragment réutilisable de contenu MDX stocké dans le répertoire `snippets/`. Importez un snippet dans n'importe quelle page pour partager du contenu sans le dupliquer. Consultez [Snippets réutilisables](/essentials/reusable-snippets).

## Référence d'API

Documentation générée automatiquement pour les endpoints de votre API à partir d'une spécification OpenAPI. Chaque endpoint devient une page interactive où les utilisateurs peuvent examiner les paramètres et tester les requêtes. Consultez [Documentation d'API](/api-reference/introduction).
