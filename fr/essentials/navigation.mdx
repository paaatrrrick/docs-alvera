Le menu de navigation est la liste des liens présente sur chaque page du site.

Vous mettrez probablement à jour `docs.json` à chaque ajout d'une nouvelle page. Les pages n'apparaissent pas automatiquement.

## Syntaxe de la navigation

Notre syntaxe de navigation est récursive, ce qui signifie que vous pouvez créer des groupes de navigation imbriqués. Vous n'avez pas besoin d'inclure `.mdx` dans les noms de page.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Dossiers

Placez simplement vos fichiers MDX dans des dossiers et mettez à jour les chemins dans `docs.json`.

Par exemple, pour avoir une page à `https://yoursite.com/your-folder/your-page`, vous créeriez un dossier nommé `your-folder` contenant un fichier MDX nommé `your-page.mdx`.

<Warning>

Vous ne pouvez pas utiliser `api` comme nom de dossier sauf si vous l'imbriquez dans un autre dossier. Mintlify utilise Next.js qui réserve le dossier `api` de premier niveau aux appels serveur internes. Un nom de dossier comme `api-reference` serait accepté.

</Warning>

```json Navigation With Folder
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Group Name",
          "pages": ["your-folder/your-page"]
        }
      ]
    }
  ]
}
```

## Pages masquées

Les fichiers MDX non inclus dans `docs.json` n'apparaîtront pas dans la barre latérale, mais restent accessibles via la barre de recherche et en liant directement vers eux.
