This glossary defines terms you'll encounter across the docs.

## MDX

A file format that combines Markdown with JSX, letting you embed interactive components directly inside your content. All pages in this documentation are authored as `.mdx` files.

## Frontmatter

The YAML block at the top of each MDX file, delimited by `---`. It sets page-level metadata such as `title`, `description`, and `icon` that Mintlify uses to render the page.

## Navigation

The structure defined in `docs.json` that organizes pages into tabs, groups, and languages. Navigation controls the sidebar, header, and routing of your docs site.

## Snippet

A reusable piece of MDX content stored in the `snippets/` directory. Import a snippet into any page to share content across the docs without duplicating it. See [Reusable snippets](/essentials/reusable-snippets).

## API reference

Auto-generated documentation for your API endpoints, sourced from an OpenAPI specification. Each endpoint becomes an interactive page where users can inspect parameters and try requests. See [API documentation](/api-reference/introduction).
