## Overview

The projects section displays a list of side projects on the personal website's home page. Each entry is rendered from the `myProjects` array and shows a title, a short description (one paragraph or a bulleted list), an image or video preview, and one or more outbound links such as GitHub, a live website, or a Chrome extension listing.

This page documents recent additions to that list so visitors and contributors know what changed and how new entries are configured.

## Entry schema

Each entry in the `myProjects` array supports the following fields:

| Field     | Type    | Required | Description                                                                                       |
| --------- | ------- | -------- | ------------------------------------------------------------------------------------------------- |
| `title`   | string  | Yes      | Project name shown as the entry header. Emoji prefixes are common.                                |
| `text`    | string \| string[] | Yes | One paragraph, or an array where the first item is a summary and the rest render as bullet points. |
| `image`   | string  | Yes      | Public URL of a preview image or video.                                                           |
| `isVideo` | boolean | Yes      | `true` if `image` points to a video file, `false` for a still image.                              |
| `links`   | array   | Yes      | List of `{ text, href }` link objects. Common labels are `Github`, `Website`, and `Play`.         |

### Bulleted descriptions

When `text` is an array, the first item is rendered as a lead paragraph and any subsequent items beginning with `- ` render as a bulleted feature list. Use this shape when a project has multiple distinct capabilities to highlight.

### Live demo links

Use a `Play` link (or `Website`) for any project with a hosted demo so visitors can try it without leaving the page. Realtime Chess uses both `Github` and `Play`.

## Adding a new entry

Open `src/constants/projects.js` and prepend a new object to the `myProjects` array. New entries appear at the top of the list on the home page.

```js src/constants/projects.js
const myProjects = [
    {
        title: '📅 AI Calendar Assistant',
        text: [
            'A calendar that reads your inbox and proactively suggests when to schedule, reschedule, or decline.',
            '- LLM-powered intent extraction from email threads',
            '- One-click Google Calendar writes via OAuth',
            '- Local-first storage, no thread content ever leaves your laptop',
        ],
        image: 'https://res.cloudinary.com/.../preview.png',
        isVideo: false,
        links: [
            { text: 'Github', href: 'https://github.com/paaatrrrick/ai-calendar' },
        ],
    },
    {
        title: '♟️ Realtime Chess',
        text: [
            'Multiplayer chess with no turns — both players move whenever they want, with a per-piece cooldown timer.',
            '- Built with React and a Node WebSocket server',
            '- Custom matchmaking + ELO',
            '- Spectator mode with delayed board state',
            '- Replays exportable as shareable PGN',
            '- Daily puzzle generated from the most chaotic real match of the previous day',
        ],
        image: 'https://res.cloudinary.com/.../preview.png',
        isVideo: false,
        links: [
            { text: 'Github', href: 'https://github.com/paaatrrrick/realtime-chess' },
            { text: 'Play', href: 'https://realtime-chess.app' },
        ],
    },
    // ...existing entries
]

export default myProjects
```

<Tip>
Set `isVideo: true` when `image` points to a `.mov` or `.mp4` file so the card renders an inline player instead of an `<img>`.
</Tip>

## Writing project descriptions

Use the `text` field to communicate what the project does and why it stands out.

- Lead with a one-sentence summary that names the core mechanic or value.
- Follow with bullets for distinct features, technical choices, or measurable wins.
- Mention the stack only when it is part of the story (for example, a WebSocket server for realtime gameplay).
- Keep each bullet short — one phrase, not a sentence.

## When to update this list

Add a new entry whenever you ship a project you want to highlight. Order entries so the most recent or most relevant project appears first; older projects can stay in place or be commented out in the source file when they no longer reflect current work.

<Note>
Project entries whose `href` is an internal path (starting with `/`) are also picked up automatically by the site's sitemap. See [Blog feeds: RSS and sitemap](/feeds) for what's included.
</Note>
