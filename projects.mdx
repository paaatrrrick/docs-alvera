## Overview

The personal website now has a dedicated `/projects` page that showcases side projects in a responsive grid. Each card shows the project name, a one-line tagline, a longer description, the tech stack as tags, and a link to the repository or live site.

Entries are sourced from the `projects` array in `src/constants/projects.js` and rendered by the `Projects` component at the `/projects` route. The page sits alongside the home page so visitors can browse the full project list without scrolling the landing page.

## When to use this page

Add an entry to `/projects` when you want to surface a side project on its own card with a tagline and tech tags. Use the home page list for projects you want featured front-and-center; use `/projects` for the complete catalog.

## Entry schema

Each object in the `projects` array supports the following fields. All fields are required.

| Field         | Type     | Description                                                                                  |
| ------------- | -------- | -------------------------------------------------------------------------------------------- |
| `name`        | string   | Project name rendered as the card heading and link text.                                     |
| `tagline`     | string   | One-line summary shown directly under the name.                                              |
| `description` | string   | Longer paragraph that explains what the project does and who it's for.                       |
| `tech`        | string[] | Stack tags rendered as a horizontal list (for example, `['React', 'Rust', 'SQLite']`).       |
| `status`      | string   | Project lifecycle. Use `'Active'` for projects you still maintain or `'Archived'` for ones you no longer ship updates to. Rendered as a colored badge in the card header. |
| `link`        | string   | URL the card heading links to. Opens in a new tab. Use the repository or live site.          |

## Adding a project

Open `src/constants/projects.js` and add a new object to the `projects` array. Entries render in array order, so put the most recent or most relevant project first.

```js src/constants/projects.js
const projects = [
    {
        name: 'Inkwell',
        tagline: 'A markdown-first note-taking app',
        description: 'A local-first notes app with bidirectional links, a command palette, and end-to-end encrypted sync.',
        status: 'Active',
        tech: ['React', 'Rust', 'SQLite'],
        link: 'https://github.com/example/inkwell',
    },
];

export default projects;
```

<Tip>
Keep `tagline` short enough to fit on one line on mobile — aim for under 60 characters. Save the longer pitch for `description`.
</Tip>

## Search and filtering

The `/projects` page includes a search box above the grid. It filters the visible cards by matching the trimmed, lowercased query against each project's `name`, `tagline`, and `description`, and combines with the tech filter (both must pass). When no projects match the current search and filter, the grid is replaced by a `No projects match your search.` empty state.

## Writing guidance

Write `name`, `tagline`, and `description` with the words a visitor would actually type — product category, problem space, or stack keywords that aren't already tags — so projects surface from search.

- **Name:** Use the canonical project name. Skip emoji prefixes; the card design relies on a clean type hierarchy.
- **Tagline:** Lead with the category or core mechanic ("Self-hosted bookmark manager", not "A project I built").
- **Description:** One or two sentences. Mention what it does and the audience or use case.
- **Tech tags:** List the load-bearing pieces of the stack, not every dependency. Three to five tags reads best.
- **Status:** Set `'Active'` for projects you still maintain. Switch to `'Archived'` once you stop shipping updates so visitors know the project is frozen — the badge turns gray instead of green.
- **Link:** Prefer the repository URL for open-source work, or the live site for hosted projects.

## Navigation

The page is wired up in `src/App.js` as:

```js src/App.js
<Route exact path='/projects' component={Projects} />
```

Link to it from anywhere in the site with `/projects`. The page includes a "← Back home" link in its header for visitors who land directly on the route.
